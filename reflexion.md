Question 1 :

Le projet est structuré avec une branche principale `master` et une branche distincte pour chaque élève (ex: `Mazine`, `Matteo`, `Lucas`). 
Chacun a commité ses modifications sur sa propre branche. L'historique montre des divergences là où les élèves ont travaillé en parallèle, puis des points de rencontre via les commandes `merge` qui rassemblent le travail.

Question 2 :

git fetch : Télécharge les infos du serveur sans toucher à vos fichiers. C'est pour "regarder".
git pull : Télécharge ET fusionne directement les changements dans vos fichiers. C'est pour "mettre à jour".

Question 3 :

git reset : Efface l'historique pour revenir en arrière (pas bien).
git revert : Crée un nouveau commit qui annule les erreurs (bien).
